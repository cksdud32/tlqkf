응용 프로그래밍/화면 구현 — 2학년 2학기 기말고사 (정리)

화면구현 문항 1~7 객관식, 8~11 서술형 / 응용 프로그래밍 서술형

1. 테스트 평가 종류 - 휴리스틱 평가: 디자인 전문가들이 사용성 원칙 등에 비추어 평가하려는 대상의 문제점을 발견하고 디자인에 반영하는 방법 - 페이퍼 프로토타입 평가: 살재 출시된 제품의 디자인을 종이 위에 미리 경험해봄으로서 수정 보완해야 할 부분은 발견하는 기법 - 선호도 평가: 사용자의 감성을 제대로 읽어내기 위해 과학적인 시점에서 객관적으로 해석 - 성능 평가: 개발 마지막 단계에서 각 제품이나 서비스의 태스크들이 지닌 장단범을 파악하기 위해 실행
2. 테스트 기법 종류 - 상황 전이 다이어그램: 하나의 객체에 대해 객체 내부의 자세한 행동을 기술하거나 시스템 전체에 대해서 자세한 행동을 기술하는데 이용하는 상호작용 다이어그램 - 심층 인터뷰: 소수의 개별 사용자에 대한 깊이 있는 정보 수집 - 포커스 그룹 인터뷰: 특정 주제와 관련하여 대상자들의 감정, 태도, 생각 등을 파악 - 맥락적 인터뷰: 인터뷰 대상자에 대한 통찰, 서브스를 둘러싼 사회적 물리적 환경을 이해함(실제 상호작용하고 있는 환경에서 인터뷰) - 파일럿 테스트: 대규모 프로젝트 실행 전 발생할 수 있는 여러 변수들을 미리 파악하여 수정, 보완하기 위해 소규모로 시험 작공해 봄.
3. 관련 용어 - 옵저베이션: 사실을 있는 그대로 확인하는 것. 실험처험 대상이나 과정에 인위적 간섭을 가하지 않는 것을 의미 - NEM 기법: 대상 제품을 처음 사용해 보는 초심자와 대상 제품 설계 및 개발에 팜여한 숙련자를 사용성 테스트에 참여시켜 태스크 수행시간을 비교하여 문제점을 객관저긍로 제시하는 기법
4. 사용성 테스트 구성 - 사용성 테스트 개요: 사용성 테스트에 대한 개요 작성 - 사용성 테스트 목적: 과제에 포함되어야 하는 주요 이슈를 정리하여 작성 - 사용자 프로파일: 어떤 사용자를 사용성 테스트에 참여시킬지에 대해 작성 - 사용성 테스트 디자인: 사용성 테스트를 어떻게 실시할 것인지에 개한 방법론 작성 - 결과 요약: 결과를 취함 방법, 결과 데이터 내용관련 기술 - 개선안 보고: 무엇을 기초로 작성할 것인지, 어떤 내용 포함 할것인지
5. ISO/IEC 9126 품질 특성 - 기능성: 명시된 요구와 내재된 여구를 만족하는 기능을 제공하는 소프트웨어 제품의 능력 - 신뢰성: 명시된 조건에서 사용될 때 성능 수준을 유지할 수 있는 소프트웨어 제품의 능력 - 사용성: 명시된 조건에서 사용 될 때 사용자에 의해 이해되고 학습, 사용, 선호될 수 있는 제품의 능력 - 효율성: 명시된 조건에서 사용되는 자원의 양에 따라 요구된 성능을 제공하는 소프트웨어 제품의 능력 - 이식성: 한 환경에서 다른 환경으로 전이될 수 있는 소프트웨어 제품의 능력 - 유지보수성: 소프트우ㅐ오 재품이 변경되는 능력(수정, 개선, 개작 등 포함)
6. UI 개선 방안 절차 - 사용성 테스트를 통해 도출된 이슈사항 이해 → UI 개선방안 수립 준비 → 개선 방안 수립 → 수정 계획 수립 -이슈 사항의 근본 원인 파악 -개선방안 수립 목적 정의 -개인적 이슈사항의 개선방안 수립 -개선방안 보고서 작성 및 검토 -이슈사항들 간의 연관성 파악 -개선방안 수립 참여 인원 구성 -연관성 있는 이슈사항 개선방안 수립 -고객에서 개선방안 설명
7. 메뉴 만들기 핵심 HTML mainnav(상위), subnav(하위) 구조 사용. CSS subnav는 기본 숨김, hover로 표시. jQuery slideDown / slideUp으로 서브메뉴 표시.
8. 휴리스틱 평가 요약 - 장점: 상대적으로 비용이 적게 들어감., 초기에 문제점을 발견할수 있음. - 단점: 구체적이고 계량적인 평가자료를 만들기 힘듦.
9. 와이어프레임 / 프로토타입 - 와이어프레임:웹 서비스 설게의 첫단계로 레이아웃을 시각적으로 나타내는 설계도 - 프로토타입: 실제와 비슷하게 구현된 시스템 초기 모델
10. 테스트 환경 구축 (모든 요소 앞에 테스트로 시작) - 목표 설정 → 항목 정의 → 참여자 확보 → 룸 설정 → 환경 설정 → 인원 구성
11. 인터뷰 방식 - 심층 인터뷰: 1대1로 진행되며 소수의 개별 사용자에 대한 깊이 있는 전보 수집을 목적으로 함. - 포커스 그룹: 다수를 대상으로 하며, 특정 주제와 관련한 대상자들의 반응을 파악하는 것을 목적으로 함.
